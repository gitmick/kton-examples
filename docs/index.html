kton examples

Small, self-contained examples for the kton substrate (plankton + nekton). Every one both creates records and uses them, and renders the resulting graph in the viewer so you see exactly what came out.

An environment is just a directory. A plankton registry lives in a folder; you point at it with PLANKTON_DIR=<dir>. A different folder is a different, independent registry. Records move between environments by content hash via plankton mirror - no server, no shared folder. Each example makes this explicit in its run.sh.
Run locally: build the kton binaries into bin/ (see the repo README), then bash examples/<name>/run.sh. The graphs below are pre-generated snapshots.
